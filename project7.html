Detecting outlier pay events

A project involving a data-cleaning pipelines in Spark, feature engineering, Scikit-Learn's IsolationForest model, and an explainer function.

To

Objective. Detect anomalous (multivariate) pay events and provide a clear explanation to the user as to what element (or group of elements) in that pay event caused the anomaly -- for example, "Social security contribution is higher than gross salary".

Approach. In this project, I used a combination of rules, percentiles, and IsolationForest to identify unusual pay patterns and amounts. One challenge I faced was sparsely (for example, bonuses and commissions). I needed to include these features in the same model as non-sparse pay types (e.g., gross salary)
